# 教练端模块文档

> 路由前缀：`/coach`
> 布局组件：`CoachLayout.vue`（深绿色侧边栏）

---

## 模块一：学员管理 - 列表（Students）

**路由**：`/coach/students`（登录后默认落地页）

**功能**
- 展示该教练名下所有学员的基本体测数据（昵称/目标/体重/BMI/体脂率/最近训练日期）
- 点击进入学员详情页

**核心业务**
- 仅返回与该教练绑定的学员，不展示其他教练的学员
- 学员与教练的绑定关系在用户预约课程或管理员分配时建立

**后端接口**
```
GET /coach/students  → 该教练名下学员列表（按 coach_id 过滤）
```

---

## 模块二：学员管理 - 详情与制定计划（StudentDetail）

**路由**：`/coach/students/:id`

**功能**
- 左侧：学员体测数据详情 + 历史训练记录表格
- 右侧：制定训练计划表单（计划名称/目标/开始结束日期/计划内容）

**核心业务**
- 教练可为学员制定个性化训练计划，计划内容存为文本（后续可扩展为结构化 JSON，按天安排具体动作）
- 一个学员可有多个计划，当前进行中的计划在用户端数据看板展示

**后端接口**
```
GET /coach/students/:id                                          → 学员详情
POST /coach/plan { user_id, title, goal, start_date, end_date, content }  → 制定计划
```

## 模块三：课程管理（CourseManage）

**路由**：`/coach/courses`

**功能**
- 展示自己发布的所有课程（名称/类型/时间/价格/名额/状态）
- 发布新课程（Dialog 弹窗表单）
- 编辑已有课程信息
- 上架/下架切换

**核心业务**
- 课程状态：1上架（用户可见可预约）/ 0下架（用户不可见）
- 下架后已预约的学员不受影响，仅新预约入口关闭
- 新建和编辑复用同一个 Dialog，通过 `form.id` 是否存在区分操作

**后端接口**
```
GET /courses/mine              → 我发布的课程列表
POST /courses                  → 发布新课程
PUT /courses/:id               → 编辑课程
PUT /courses/:id/status        → 上架/下架
```

## 模块四：内容投稿 - 动作（ActionPublish）

**路由**：`/coach/publish/action`

**功能**
- 左侧：填写动作信息（名称/部位/类型/难度/视频URL/简介/注意事项）并提交审核
- 右侧：查看自己已投稿的动作列表及审核状态

**核心业务**
- 教练投稿后状态为"待审核（0）"，管理员审核通过后变为"已上线（1）"，用户才能看到
- 审核状态：0待审核 / 1已上线 / 2已驳回 / 3已下架
- 驳回时管理员填写原因，后续对接消息通知功能推送给教练
- 视频目前填写 URL，后续替换为 `el-upload` 上传至 OSS

**后端接口**
```
POST /actions       → 发布动作（status 自动设为 0）
GET /actions/mine   → 我投稿的动作列表
```

---

## 模块五：内容投稿 - 文章（ArticlePublish）

**路由**：`/coach/publish/article`

**功能**
- 左侧：填写文章信息（标题/分类/封面图URL/正文）并提交审核
- 右侧：查看自己已投稿的文章列表及审核状态

**核心业务**
- 与动作投稿相同的审核流程，状态机一致
- 文章正文目前为 textarea，后续替换为 `vue-quill-editor` 富文本编辑器
- 封面图目前填写 URL，后续替换为 `el-upload` 上传至 OSS

**后端接口**
```
POST /diet/articles       → 发布文章（status 自动设为 0）
GET /diet/articles/mine   → 我投稿的文章列表
```
