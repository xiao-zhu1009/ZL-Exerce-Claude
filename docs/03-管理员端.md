# 管理员端模块文档

> 路由前缀：`/admin`
> 布局组件：`AdminLayout.vue`（深紫色侧边栏）

---

## 模块一：数据统计（Statistics）

**路由**：`/admin/statistics`（登录后默认落地页）

**功能**
- 6张概览卡片：总用户数、活跃用户、动作总数、文章总数、课程总数、预约总数
- 用户增长趋势面积折线图（近6个月）

**核心业务**
- 全平台数据的汇总视图，帮助管理员掌握平台整体运营状况
- 图表使用 ECharts@4 渲染

**后端接口**
```
GET /admin/statistics  → 全局统计数据（含用户增长数组）
```

---

## 模块二：用户管理（UserManage）

**路由**：`/admin/users`

**功能**
- 用户列表：支持关键词（用户名/昵称）搜索和角色筛选
- 封禁/解封用户账号
- 角色升降：普通用户 ↔ 教练

**核心业务**
- 封禁后用户无法登录（后端 JWT 验证时检查 status 字段）
- 角色升级为教练后，该用户可访问教练端功能（前端路由守卫根据 role 判断）
- 角色降级时，该教练名下学员和课程的处理策略由后端业务决定

**后端接口**
```
GET /admin/users?keyword=&role=       → 用户列表
PUT /admin/users/:id/status { status }  → 封禁/解封
PUT /admin/users/:id/role { role }      → 修改角色
```

## 模块三：内容审核 - 动作（ActionReview）

**路由**：`/admin/review/actions`

**功能**
- 展示所有待审核（status=0）的动作投稿列表
- 一键通过：动作状态变为已上线（1），用户端可见
- 驳回：弹窗填写驳回原因，动作状态变为已驳回（2），后续通知教练

**核心业务**
- 审核是内容质量的核心把关环节，通过后内容才对用户可见
- 驳回原因存入 `reject_reason` 字段，后续消息通知功能将推送给投稿教练
- 审核操作后该条目从列表移除（乐观更新）

**后端接口**
```
GET /admin/actions/pending                              → 待审核动作列表
PUT /admin/actions/:id/review { status, reject_reason? }  → 审核（通过/驳回）
```

## 模块四：内容审核 - 文章（ArticleReview）

**路由**：`/admin/review/articles`

**功能**
- 展示所有待审核（status=0）的饮食文章投稿列表
- 一键通过：文章状态变为已上线（1），用户端可见
- 驳回：弹窗填写驳回原因，文章状态变为已驳回（2）

**核心业务**
- 与动作审核流程完全一致，复用相同的状态机设计
- 审核通过的文章在用户端饮食中心文章列表中展示

**后端接口**
```
GET /admin/diet/pending                              → 待审核文章列表
PUT /admin/diet/:id/review { status, reject_reason? }  → 审核（通过/驳回）
```

---

## 内容审核状态机

```
教练投稿
   ↓
待审核（0）
   ↓         ↓
通过（1）   驳回（2）
   ↓
已上线，用户可见
   ↓
管理员可强制下架（3）
```
