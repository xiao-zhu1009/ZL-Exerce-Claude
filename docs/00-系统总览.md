# 智能健身系统 - 系统总览

> 技术栈：Vue2 + Element-UI + FastAPI + MySQL

---

## 一、三端权限关系

```
┌─────────────────────────────────────────────────────┐
│                   超级管理员（admin）                  │
│  · 审核教练投稿的内容（动作/文章）                      │
│  · 管理所有用户账号（封禁/角色升降）                    │
│  · 查看全平台数据统计                                  │
└───────────────────┬─────────────────────────────────┘
                    │ 审核通过后内容对用户可见
                    │ 可将用户升级为教练
                    ▼
┌─────────────────────────────────────────────────────┐
│                    教练（coach）                      │
│  · 向平台投稿动作库内容和饮食文章（需管理员审核）        │
│  · 管理名下学员，为学员制定训练计划                     │
│  · 发布和管理自己的课程                                │
└───────────────────┬─────────────────────────────────┘
                    │ 发布课程供用户预约
                    │ 为绑定学员制定计划
                    │ 投稿内容审核通过后用户可浏览
                    ▼
┌─────────────────────────────────────────────────────┐
│                   普通用户（user）                     │
│  · 浏览动作库和饮食文章（仅已审核通过内容）              │
│  · 记录每日饮食，自动计算营养摄入                       │
│  · 预约教练发布的课程                                  │
│  · 查看教练为自己制定的训练计划                         │
│  · 在数据看板查看自身健康趋势                           │
└─────────────────────────────────────────────────────┘
```

---

## 二、核心数据流

```
教练投稿动作/文章
      ↓
管理员审核（通过/驳回）
      ↓
内容上线，用户可浏览学习
      ↓
用户记录饮食 + 训练打卡
      ↓
数据看板展示健康趋势
```

---

## 三、权限隔离策略

| 层级 | 实现方式 |
|------|---------|
| 前端路由 | `router/index.js` 中每个路由组配置 `meta.role`，`beforeEach` 守卫拦截非法访问 |
| 前端菜单 | 三端使用独立布局组件（UserLayout / CoachLayout / AdminLayout），天然隔离 |
| 后端接口 | JWT Token 解析角色，依赖注入中间件拦截（对接后端时实现） |
