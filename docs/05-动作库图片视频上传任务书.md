# 【任务书】动作库图片/视频上传全流程完善

**日期：** 2026-04-12  
**涉及模块：** 教练端投稿 / 管理员端审核 / 用户端查看

---

## 1. 需求说明

当前动作投稿表单中，封面图和视频仅支持手填 URL，无法实际上传文件。  
需要完善为：
- 教练端：可上传封面图（图片文件）+ 视频文件，替换原来的文本输入框
- 管理员端：审核预览弹窗中展示封面图和视频播放器
- 用户端：动作详情页已有视频播放器，补充封面图展示；动作列表卡片展示封面图缩略图

---

## 2. 实现思路

### 文件存储方案（本地存储，复用现有方案）

后端已有 `static/` 目录通过 `/api/static/` 对外暴露（头像上传已验证可用）。  
新增两个子目录：
- `static/action_covers/` — 封面图（JPG/PNG/WebP，限 5MB）
- `static/action_videos/` — 视频文件（MP4/WebM，限 200MB）

文件命名规则：`{coach_id}_{uuid}.{ext}`（与头像命名一致）

前端访问 URL：`http://127.0.0.1:8008/api/static/action_covers/xxx.jpg`

### 上传流程

```
教练选择文件 → el-upload 组件 → 调用独立上传接口 → 后端保存文件返回相对路径
→ 前端将路径存入 form.cover_img / form.video_url → 提交动作表单时一起带上
```

上传和投稿分两步，避免大文件混在 JSON 请求里。

## 4. 后端要做什么

### 4.1 新增两个文件上传接口（api/coach_action.py）
```
POST /coach/actions/upload/cover  — 上传封面图，返回相对路径
POST /coach/actions/upload/video  — 上传视频，返回相对路径
```

### 4.2 创建静态资源目录
```
static/action_covers/   （启动时自动创建）
static/action_videos/   （启动时自动创建）
```

### 4.3 文件校验规则
| 类型 | 允许格式 | 大小限制 |
|------|---------|---------|
| 封面图 | JPG/PNG/WebP | 5MB |
| 视频 | MP4/WebM | 200MB |

### 4.4 数据库表无需变更
`actions` 表已有 `cover_img` 和 `video_url` 字段（VARCHAR 500），存相对路径即可。

---

## 5. 接口设计

### 上传封面图
```
POST /api/coach/actions/upload/cover
Content-Type: multipart/form-data
Body: file (UploadFile)
Auth: Bearer Token（coach 角色）

Response:
{
  "code": 200,
  "msg": "上传成功",
  "data": { "path": "action_covers/1_abc123.jpg" }
}
```

### 上传视频
```
POST /api/coach/actions/upload/video
Content-Type: multipart/form-data
Body: file (UploadFile)
Auth: Bearer Token（coach 角色）

Response:
{
  "code": 200,
  "msg": "上传成功",
  "data": { "path": "action_videos/1_abc123.mp4" }
}
```

### 已有接口无需改动
- `POST /coach/actions` — ActionCreate schema 已有 cover_img / video_url 字段
- `PUT /coach/actions/{id}` — ActionUpdate schema 已有对应字段
- `GET /actions/{id}` — 已返回 cover_img / video_url

---

## 6. 数据表结构

无需变更，现有字段已满足：
```sql
cover_img  VARCHAR(500)  -- 存相对路径，如 action_covers/1_abc.jpg
video_url  VARCHAR(500)  -- 存相对路径，如 action_videos/1_abc.mp4
```

前端拼接完整 URL：`BASE_URL + '/static/' + path`
