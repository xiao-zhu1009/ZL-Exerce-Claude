# 【任务书】饮食中心与饮食记录模块合并

## 1. 需求说明
将侧栏中「饮食中心」与「饮食记录」两个独立入口合并为单一「饮食中心」模块，通过 Tab 切换子功能，消除用户在两个相近页面间来回跳转的割裂感。

## 2. 实现思路
- `DietCenter/index.vue` 新增「饮食记录」Tab，将 `DietRecord.vue` 的全部逻辑（每日详情 + 近期趋势 + 弹窗）内联进来
- 删除顶部「我的饮食记录」跳转按钮
- 路由：保留 `diet/record` 路由但改为重定向到 `/user/diet?tab=record`，避免旧收藏失效
- 侧栏：去掉「饮食记录」菜单项，仅保留「饮食中心」
- `DietRecord.vue` 文件保留但不再挂载路由（可后续删除）

## 3. 前端要做什么
1. 改造 `DietCenter/index.vue`：新增「饮食记录」Tab，合并 DietRecord 的 data/computed/methods/template/style
2. 支持 URL query `?tab=record` 进入时自动激活对应 Tab
3. 修改 `UserLayout.vue`：删除「饮食记录」菜单项
4. 修改 `router/index.js`：`diet/record` 改为重定向

## 4. 后端要做什么
无需改动，接口不变。

## 5. 接口设计
复用现有接口，无新增：
- `GET /diet/records?record_date=`
- `GET /diet/records/range`
- `POST /diet/records`、`PUT /diet/records/{id}`、`DELETE /diet/records/{id}`

## 6. 数据表结构
无变更。
