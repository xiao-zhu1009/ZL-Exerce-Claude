# 用户端模块文档

> 路由前缀：`/user`
> 布局组件：`UserLayout.vue`（深蓝色侧边栏）

---

## 模块一：数据看板（Dashboard）

**路由**：`/user/dashboard`（登录后默认落地页）

**功能**
- 4张概览卡片：本周训练次数、本周消耗卡路里、今日摄入卡路里、当前体重
- 体重变化折线图：展示历次录入的体重趋势
- 本周训练频次柱状图：按周一至周日展示每日训练时长

**核心业务**
- 汇聚用户的运动和饮食数据，形成健康数据闭环
- 图表使用 ECharts@4 渲染，数据来源于身体记录表和训练打卡记录

**后端接口**
```
GET /user/dashboard     → 概览数据 + 本周训练记录
GET /user/body-records  → 体重历史（折线图数据源）
```

## 模块二：动作库（ActionLibrary）

**路由**：`/user/actions`（列表）、`/user/actions/:id`（详情）

**功能**
- 列表页：卡片展示所有已上线动作，支持训练部位/动作类型/难度/关键词四维筛选
- 详情页：教学视频（预留）、图文步骤（el-steps 展示）、注意事项

**核心业务**
- 仅展示管理员审核通过（status=1）的动作内容
- 筛选条件实时过滤，无需翻页重载
- 浏览详情时后端触发 `view_count+1`

**后端接口**
```
GET /actions?body_part=&category=&difficulty=&keyword=  → 动作列表
GET /actions/:id                                        → 动作详情（+浏览量）
```

## 模块三：饮食中心 - 文章（DietCenter）

**路由**：`/user/diet`（列表）、`/user/diet/articles/:id`（详情）

**功能**
- 文章列表：按分类（增肌餐/减脂餐/均衡饮食/补剂知识）和关键词筛选
- 文章详情：富文本内容渲染（`v-html`）

**核心业务**
- 仅展示管理员审核通过（status=1）的文章
- 文章正文为富文本 HTML，前端直接 `v-html` 渲染

**后端接口**
```
GET /diet/articles?category=&keyword=  → 文章列表
GET /diet/articles/:id                 → 文章详情
```

## 模块四：饮食中心 - 饮食记录（DietRecord）

**路由**：`/user/diet/record`

**功能**
- 按日期查看当天饮食记录
- 食物搜索联想（el-autocomplete），选中后自动填入营养数据
- 按食用量比例自动换算卡路里/蛋白质/碳水/脂肪
- 按餐次（早/午/晚/加餐）分组展示，每餐小计热量
- 页面顶部实时汇总全天营养摄入

**核心业务**
- 营养换算公式：`实际营养 = 基准营养(per 100g) × 食用量 / 100`，纯前端计算
- 饮食记录是数据看板"今日摄入卡路里"的数据来源

**后端接口**
```
GET /diet/foods/search?keyword=  → 食物搜索
GET /diet/records?date=          → 当日记录列表
POST /diet/records               → 添加记录
DELETE /diet/records/:id         → 删除记录
```

## 模块五：课程预约（Course）

**路由**：`/user/courses`

**功能**
- 展示所有上架课程（教练/时间/价格/剩余名额）
- 满员课程自动禁用预约按钮
- 预约成功后实时更新名额显示

**核心业务**
- 名额控制：前端展示 `enrolled/max_people`，后端需用行锁防止并发超额
- 预约记录关联用户和课程，状态：0待确认/1已确认/2已取消

**后端接口**
```
GET /courses                        → 课程列表
POST /reservations { course_id }    → 预约课程
DELETE /reservations/:id            → 取消预约
```

## 模块六：个人主页（Profile）

**路由**：`/user/profile`

**功能**
- 编辑基本信息：昵称、手机号、训练目标（减脂/增肌/塑形/维持）
- 录入身体数据：身高、体重、体脂率，BMI 自动计算并用颜色标注健康状态
- 历史身体数据表格

**核心业务**
- BMI 计算：`weight / (height/100)²`，实时计算
- BMI 健康状态：<18.5 偏瘦（蓝）/ 18.5-24 正常（绿）/ 24-28 超重（橙）/ ≥28 肥胖（红）
- 身体数据历史是数据看板体重折线图的数据来源

**后端接口**
```
GET /user/profile          → 个人信息
PUT /user/profile          → 更新信息
GET /user/body-records     → 历史身体数据
POST /user/body-record     → 新增身体数据
```
