# 【任务书】训练中心改造

## 1. 需求说明
- 用户端：可自己添加训练记录（已有）、自己创建训练计划（新增）、查看所有训练计划含教练指定的（已有，扩展）
- 教练端：在学员详情页为指定学员制定训练计划，对接真实后端接口（当前全是 mock）

## 2. 实现思路
- 用户端 TrainingPlan.vue 加"新建计划"入口，弹窗填写后 POST /training/plans
- 教练端 coach.js 替换 mock → 真实 axios 请求
- 教练端 Detail.vue 完善表单（加 description、表单校验、展示该学员已有计划列表）
- 后端新增教练专属接口：查学员列表、查学员详情（含训练记录）、为学员创建计划

## 3. 前端要做什么
1. `src/api/training.js`：加 `addTrainingPlan` (POST /training/plans)
2. `src/api/coach.js`：替换 mock，对接 /coach/students、/coach/students/:id、/coach/training/plans
3. `src/views/user/Training/TrainingPlan.vue`：加新建计划弹窗
4. `src/views/coach/Students/Detail.vue`：完善制定计划表单（校验、description、已有计划展示）

## 4. 后端要做什么
1. `schemas/training.py`：加 TrainingPlanCreate schema
2. `CRUD/training.py`：加 create_training_plan、get_coach_students、get_student_detail_for_coach
3. 新建 `api/coach_training.py`：教练端接口
4. `main.py`：注册 coach_training_router

## 5. 接口设计

### 用户端
| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /training/plans | 用户自建训练计划 |

### 教练端
| 方法 | 路径 | 说明 |
|------|------|------|
| GET  | /coach/students | 查询该教练名下学员列表 |
| GET  | /coach/students/:id | 查询学员详情（含最近训练记录） |
| POST | /coach/training/plans | 为学员创建训练计划 |
| GET  | /coach/training/plans?student_id= | 查询某学员的训练计划列表 |

## 6. 数据表结构
复用已有 training_plans 表，coach_id 字段区分是教练创建还是用户自建（用户自建时 coach_id=0）
