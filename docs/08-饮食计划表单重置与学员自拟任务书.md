【任务书】

1. 需求说明：
   - 教练端提交饮食计划后，表单的"整体说明"字段未被清空（其他字段已重置）
   - 学员端饮食计划页面目前只能查看教练制定的计划，需新增学员自拟饮食计划功能

2. 实现思路：
   - Bug修复：`submitDietPlan` 成功后调用 `resetFields()` 但 `description` 字段未被 el-form rules 覆盖，
     导致 resetFields 不清空它；改为提交成功后手动重置整个 dietForm 对象
   - 新功能：学员端 DietPlan.vue 增加"自拟计划"Tab，包含创建表单和自拟计划列表，
     调用新增的学员自拟接口（与教练制定接口区分，coach_id=0 标识自建）

3. 前端要做什么：
   - Detail.vue（教练端）：submitDietPlan 成功后将 dietForm 整体重置为初始值，而非仅 resetFields
   - DietPlan.vue（学员端）：
     a. 顶部增加 Tab 切换：「教练计划」/ 「我的计划」
     b. 「我的计划」Tab 包含：自拟计划列表（卡片展示）+ 创建表单（计划名称、目标、日期、整体说明、每日餐次）
     c. 自拟计划支持终止/标记完成操作

4. 后端要做什么：
   - 新增学员自拟饮食计划接口（POST /diet-plans/self）
   - 新增学员自拟计划列表接口（GET /diet-plans/self）
   - 新增学员自拟计划状态更新接口（PUT /diet-plans/self/{plan_id}/status）
   - 或复用现有接口，通过 coach_id=0 区分自建计划（视后端现有实现决定）

5. 接口设计：
   POST /diet-plans/self
   Body: { title, goal, start_date, end_date, description, content: { days: [...] } }
   Response: { code: 200, msg: "创建成功", data: { id, ... } }

   GET /diet-plans/self?status=
   Response: { code: 200, msg: "ok", data: [...] }

   PUT /diet-plans/self/{plan_id}/status
   Body: { status: 0|2 }
   Response: { code: 200, msg: "ok", data: null }

6. 数据表结构：
   复用现有 diet_plans 表，coach_id=0 表示学员自建，无需新增表
