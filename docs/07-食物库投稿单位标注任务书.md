【任务书】

1. 需求说明：
   教练端食物库投稿页面，营养成分字段（热量、蛋白质、碳水、脂肪、膳食纤维）需要明确标注"每 100g / 100ml"为计量单位，让教练填写时清楚知道填的是每100单位的数值，而非每份/每粒等其他单位。后端接收和存储逻辑保持不变（数据库本身已按每100单位存储），但需确认接口文档和字段注释与此一致。

2. 实现思路：
   - 前端：在投稿表单的 label 和 placeholder 上补充"每100g/ml"说明，卡片标题已有提示但字段本身不够直观
   - 后端：schema 字段注释补充说明单位含义，无需改动数据处理逻辑（已是每100单位存储）

3. 前端要做什么：
   - FoodSubmit/index.vue：
     a. 卡片 header 标题保持"投稿新食物（每 100g / 100ml 的营养素）"
     b. 热量 label 改为"热量(kcal/100)"
     c. 蛋白质 label 改为"蛋白质(g/100)"
     d. 碳水 label 改为"碳水(g/100)"
     e. 脂肪 label 改为"脂肪(g/100)"
     f. 膳食纤维 label 改为"膳食纤维(g/100)"
     g. 各营养成分输入框加 placeholder 提示"每100g/ml的含量"
   - 投稿列表表头同步更新，让数据展示也清晰

4. 后端要做什么：
   - schemas/diet_record.py：FoodCreate 各营养字段注释补充"每100g/ml"说明
   - 数据处理逻辑无需改动，存储和返回均已是每100单位的值

5. 接口设计：
   无变更，接口字段和格式保持不变

6. 数据表结构：
   无变更，foods 表字段含义本身就是每100单位的营养素值
