## 一、模块概述

动作库是健身平台的核心内容模块，采用 **教练投稿 → 超管审核 → 用户浏览** 的三角流转机制。

### 角色职责

| 角色 | 权限 |
|------|------|
| 教练 | 发布动作（含视频/封面上传）、查看自己投稿的状态、修改被驳回的动作 |
| 超级管理员 | 审核待审动作（通过/驳回）、下架已上线动作、查看全部动作 |
| 普通用户 | 浏览已上线动作、筛选/搜索、查看详情（含视频） |

## 二、数据库表设计

### 2.1 actions 主表（已有，需补充字段）

```sql
CREATE TABLE actions (
    id            INT PRIMARY KEY AUTO_INCREMENT,
    author_id     INT NOT NULL COMMENT '教练用户ID',
    title         VARCHAR(100) NOT NULL COMMENT '动作名称',
    body_part     VARCHAR(20) NOT NULL COMMENT '训练部位：胸/背/腿/肩/手臂/核心',
    category      VARCHAR(20) NOT NULL COMMENT '动作类型：力量/有氧/拉伸/综合',
    difficulty    TINYINT NOT NULL DEFAULT 1 COMMENT '难度：1初级 2中级 3高级',
    cover_img     VARCHAR(500) NOT NULL DEFAULT '' COMMENT '封面图路径（相对路径）',
    video_url     VARCHAR(500) NOT NULL DEFAULT '' COMMENT '视频路径（相对路径）',
    description   TEXT COMMENT '动作简介',
    steps         JSON COMMENT '步骤列表 ["步骤1","步骤2"]',
    cautions      JSON COMMENT '注意事项列表 ["注意1","注意2"]',
    view_count    INT NOT NULL DEFAULT 0 COMMENT '浏览次数',
    status        TINYINT NOT NULL DEFAULT 0 COMMENT '0待审核 1已上线 2已驳回 3已下架',
    reject_reason VARCHAR(255) NOT NULL DEFAULT '' COMMENT '驳回原因',
    reviewed_by   INT COMMENT '审核管理员ID',
    reviewed_at   DATETIME COMMENT '审核时间',
    created_at    DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at    DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,

    INDEX idx_status (status),
    INDEX idx_author (author_id),
    INDEX idx_body_part (body_part),
    INDEX idx_category (category)
);
```

**status 状态流转：**
```
0(待审核) ──通过──→ 1(已上线) ──下架──→ 3(已下架)
0(待审核) ──驳回──→ 2(已驳回) ──修改重投──→ 0(待审核)
```

## 三、文件存储设计

### 3.1 存储方案选择

| 方案 | 适用场景 | 优缺点 |
|------|----------|--------|
| **本地存储（当前推荐）** | 开发/小规模部署 | 简单，无费用；不适合生产扩展 |
| 阿里云 OSS | 生产环境 | 稳定可靠，有费用 |
| 腾讯云 COS | 生产环境 | 同上 |

**当前阶段使用本地存储**，FastAPI 通过 `StaticFiles` 挂载静态目录对外提供访问。

### 3.2 本地目录结构

```
ZL_BG_FastApi/
└── static/
    └── uploads/
        ├── actions/
        │   ├── covers/          # 封面图
        │   │   └── {action_id}_{timestamp}.jpg
        │   └── videos/          # 动作视频
        │       └── {action_id}_{timestamp}.mp4
        └── avatars/             # 用户头像（已有）
```

### 3.3 文件访问 URL 规则

```
封面图：http://127.0.0.1:8008/static/uploads/actions/covers/xxx.jpg
视频：  http://127.0.0.1:8008/static/uploads/actions/videos/xxx.mp4
```

数据库中存储**相对路径**（`uploads/actions/covers/xxx.jpg`），前端拼接 baseURL 访问。

### 3.4 上传限制

| 类型 | 格式 | 大小限制 |
|------|------|----------|
| 封面图 | jpg/png/webp | ≤ 5MB |
| 视频 | mp4/mov | ≤ 200MB |

## 四、接口设计

### 4.1 用户端接口

```
GET  /api/actions                    # 获取动作列表（仅 status=1）
     ?body_part=胸&category=力量&difficulty=2&keyword=深蹲&page=1&page_size=12

GET  /api/actions/{id}               # 获取动作详情（浏览数+1）
```

### 4.2 教练端接口

```
POST   /api/coach/actions            # 发布新动作（multipart/form-data，含文件上传）
GET    /api/coach/actions            # 获取我发布的动作列表
GET    /api/coach/actions/{id}       # 获取我的某个动作详情
PUT    /api/coach/actions/{id}       # 修改被驳回的动作（重新提交审核）
DELETE /api/coach/actions/{id}       # 删除待审核/已驳回的动作
```

### 4.3 管理员端接口

```
GET  /api/admin/actions              # 获取所有动作（可按 status 筛选）
     ?status=0&page=1&page_size=20

PUT  /api/admin/actions/{id}/review  # 审核动作
     Body: { "status": 1 }           # 通过
     Body: { "status": 2, "reject_reason": "视频不清晰" }  # 驳回

PUT  /api/admin/actions/{id}/offline # 下架已上线动作
```

### 4.4 文件上传接口

```
POST /api/upload/action-cover        # 上传封面图，返回 { url: "相对路径" }
POST /api/upload/action-video        # 上传视频，返回 { url: "相对路径" }
```

### 4.5 统一响应格式


### 7.2 教练只能操作自己的动作

```python
# CRUD 层查询时加 author_id 过滤
async def get_coach_actions(db, author_id: int):
    result = await db.execute(
        select(Action).where(Action.author_id == author_id)
    )
    return result.scalars().all()
```

### 7.3 审核时记录审核人

```python
async def review_action(db, action_id: int, status: int,

    action.status = status
```
