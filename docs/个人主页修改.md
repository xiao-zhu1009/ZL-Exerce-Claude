# 【任务书】个人主页重构

## 1. 需求说明

对个人主页（`src/views/user/Profile/index.vue`）进行分区分块重构，分为两大模块：

- **账号信息区**：昵称、手机号、账号（用户名）、密码修改、个性签名
- **身体指标区**：身高、体重、BMI（自动计算）、体脂率、腰围、臀围及其他指标，支持编辑修改

当前阶段数据先写死（mock），后续从数据库接口获取。

---

## 2. 实现思路

- 页面整体采用左右两栏布局（`el-row` + `el-col`）
- 左栏：账号信息卡片，支持编辑保存
- 右栏：身体指标卡片，支持编辑保存，BMI 自动联动计算
- 数据先用 `data()` 中的静态 mock 数据，预留 `created()` 中的接口调用注释
- 密码修改使用独立的弹窗（`el-dialog`）

---

## 3. 前端要做什么

### 文件：`src/views/user/Profile/index.vue`

**账号信息区块（左侧卡片）：**
| 字段 | 类型 | 说明 |
|------|------|------|
| nickname | 文本输入 | 昵称 |
| phone | 文本输入 | 手机号 |
| username | 文本输入（只读） | 账号，不可修改 |
| signature | 文本域 | 个性签名，最多 100 字 |
| password | 按钮触发弹窗 | 点击弹出修改密码对话框 |

**身体指标区块（右侧卡片）：**
| 字段 | 类型 | 说明 |
|------|------|------|
| height | 数字输入 | 身高 (cm) |
| weight | 数字输入 | 体重 (kg) |
| bmi | 只读展示 | 自动计算，带状态标签（偏瘦/正常/超重/肥胖） |
| body_fat | 数字输入 | 体脂率 (%) |
| waist | 数字输入 | 腰围 (cm) |
| hip | 数字输入 | 臀围 (cm) |
| whr | 只读展示 | 腰臀比，自动计算（waist/hip） |

**修改密码弹窗：**
- 旧密码、新密码、确认新密码
- 前端校验新密码一致性

---

## 4. 后端要做什么（后续对接）

> 当前阶段不需要后端改动，数据写死在前端。后续需要以下接口：

| 接口 | 方法 | 说明 |
|------|------|------|
| `/user/profile` | GET | 获取账号信息 |
| `/user/profile` | PUT | 更新账号信息（昵称、手机、签名） |
| `/user/body-stats` | GET | 获取最新身体指标 |
| `/user/body-stats` | PUT | 更新身体指标 |
| `/user/change-password` | POST | 修改密码 |

---

## 5. 接口设计（后续）

### GET /user/profile
```json
{
  "code": 200,
  "data": {
    "nickname": "健身达人",
    "phone": "138****8888",
    "username": "user001",
    "signature": "每天进步一点点"
  }
}
```

### GET /user/body-stats
```json
{
  "code": 200,
  "data": {
    "height": 175,
    "weight": 70.5,
    "bmi": 23.0,
    "body_fat": 18.5,
    "waist": 80,
    "hip": 95
  }
}
```

### PUT /user/body-stats
```json
请求体：{ "height": 175, "weight": 70.5, "body_fat": 18.5, "waist": 80, "hip": 95 }
```

### POST /user/change-password
```json
请求体：{ "old_password": "xxx", "new_password": "yyy" }
```
